Section 01 · Initiation
Project Context & Charter
Background, objectives, scope, and project authorisation
HAFSA YASEEN · PM PORTFOLIO

The Inherited Challenge: The Medical Research Foundation held over 160 legacy giving records spanning more than 80 years. These records had never been comprehensively analysed. The data existed across multiple sources, in inconsistent formats, with varying levels of completeness across decades. A partial process existed but was underdeveloped and not being used strategically.

The Opportunity: Legacy giving is one of the most valuable long-term income streams for any charity. Understanding who gives, when, how much, and why — across decades of history — could fundamentally reshape how MRF targets, cultivates, and engages potential legacy donors going forward.

Hafsa's Role: Hafsa inherited this project and developed it significantly. Working with her line manager, she took ownership of the full analysis lifecycle: data collection, cleaning, standardisation, analysis, dashboard build, and strategic presentation to senior leadership. The outcome was a live Power BI dashboard and a new legacy giving strategy for MRF.

Section 03 · Data Management
Data Management & Methodology
Collection, cleaning, standardisation, and Salesforce structuring
HAFSA YASEEN · PM PORTFOLIO

The biggest technical challenge of this project was the data itself. 80+ years of legacy records meant wildly inconsistent formats, missing fields, outdated naming conventions, and records held across multiple disconnected systems. Before any analysis could begin, the data had to be painstakingly collected, cleaned, and standardised into a single coherent structure.

Data challenges
Challenge 1
Inconsistent formats
Records from different decades used different field names, date formats, currency conventions, and donor identifiers. No single source of truth existed. Each decade required bespoke cleaning rules.
Challenge 2
Incomplete records
Older records (pre-1990s) had significant gaps — missing bequest values, dates, and donor demographics. Decisions had to be made on how to handle missing data without distorting analysis.
Challenge 3
Unclear stakeholder requirements
At project start, senior leadership had broad questions but no specific brief. Hafsa worked with her manager to define the analytical scope iteratively through stakeholder conversations.
Data management methodology
Step 1: Data Collection
Identified all sources holding legacy records: Salesforce CRM, Excel files, paper archives, email records. Extracted all 160+ records into a master Excel workbook. Documented source, format, and completeness level for each record batch.
Step 2: Data Cleaning (Excel)
Standardised date formats across all records. Unified donor naming conventions. Filled missing values using available evidence (cross-referencing sources). Flagged genuinely incomplete records for management review. Created a cleaning log documenting every decision made.
Step 3: Standardised Data Structure
Designed a consistent data schema enabling analysis across all 80+ years: donor ID, year of bequest notification, bequest value, donor age bracket, giving history, bequest type. Applied to all 160+ records.
Step 4: Salesforce CRM Structuring
Entered cleaned data into Salesforce using a structured approach. Improved on the existing partial process — created consistent field usage, tagging, and record categorisation to enable ongoing CRM-based legacy tracking.
Step 5: Data Validation
Cross-referenced cleaned dataset against original sources. Reviewed with line manager. Confirmed data integrity before proceeding to analysis phase.
Tools used
Microsoft Excel
Primary data cleaning tool. Advanced formulas for standardisation. Pivot tables for initial exploration. Cleaning log maintained throughout.
Salesforce CRM
Structured entry of cleaned records. Improved tagging and field usage. Ongoing CRM foundation for future legacy tracking by the fundraising team.
Power BI
Connected to cleaned dataset. Built live dashboards for ongoing use. Visualised trends, donor profiles, and giving patterns for leadership.

Section 05 · Analysis & Deliverable
Analysis & Dashboard Deliverable
Key insights, patterns discovered, and live Power BI dashboard
HAFSA YASEEN · PM PORTFOLIO
Key insights discovered
Insight 1
Peak giving periods identified
Analysis revealed distinct decades with significantly higher legacy notification rates. These periods correlated with specific MRF campaigns and broader charitable giving trends — informing when to intensify legacy cultivation activity.
Insight 2
Typical donor profile emerged
A clear profile of the "typical" MRF legacy donor emerged from the data: age bracket, prior giving history, bequest value range, and giving frequency. This profile had never been documented before.
Insight 3
Unrecognised donor segments
Previously unidentified donor segments were discovered — groups of donors sharing characteristics who had never been treated as a distinct segment for targeting purposes. These became priority targets for the new strategy.

Lessons learned
1
Define the questions before touching the data
Starting with unclear stakeholder requirements nearly led to analysis that couldn't be acted on. Taking time to define 5 specific questions upfront gave the whole project direction and made the final recommendations immediately usable.
2
Document every data decision — it becomes your audit trail
The cleaning decisions log proved invaluable when stakeholders questioned specific data points. Being able to explain exactly why a record was handled a certain way built credibility and trust in the analysis.
3
Scope creep is especially dangerous in open-ended data projects
Without a clearly documented scope, every interesting data question becomes a potential project expansion. The scope change log and formal process kept the project focused and delivered on time.
4
A live dashboard creates more value than a one-time report
Delivering a live Power BI dashboard rather than a static report meant the analysis continued generating value long after the project closed — turning a one-off investigation into ongoing institutional intelligence.
