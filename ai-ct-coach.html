The problem

Most AI tools give you answers. That's the problem.

There's no shortage of AI that will tell you what to think, summarise arguments for you, or write your reasoning out. But critical thinking is a muscle — and you don't build it by being handed conclusions.

I wanted to build something that worked the other way: a tool that takes a position you've half-formed and systematically pulls it apart — not to win an argument, but to help you think more clearly.

"The goal isn't to change what you think. It's to make you actually know why you think it."

What I built

A Socratic coach with invisible scaffolding

The app presents real-world scenarios across six domains — or lets you bring a personal decision you're wrestling with — and then acts as a Socratic coach: never offering opinions, only asking the questions that surface your assumptions, challenge your evidence, and push you toward perspectives you haven't considered.

Underneath, the coach is silently assessing five components of critical thinking — Analysis, Evaluation, Inference, Explanation, and Self-Regulation — without ever naming them. When you request an evaluation, you get scored on each one individually, with specific feedback on where your reasoning was sharp and where it had gaps.

Design detail

Four difficulty tiers, calibrated by complexity

Each level changes not just the topic difficulty but the nature of the reasoning required. At Level 1, there are clear stakeholders and obvious trade-offs. By Level 4, you're dealing with wicked problems — no clean answers, only managed tensions.

Level Name What it demands
1 Foundation Identifying the main trade-offs; articulating a position with basic reasoning
2 Developing Holding multiple valid perspectives simultaneously; surfacing hidden assumptions
3 Advanced Historical context, systemic factors, competing values — no easy resolution
4 Expert Global implications, deep philosophical tensions, second and third-order consequences
Technical architecture

React frontend, serverless backend, zero exposed keys

The stack is intentionally simple. A React single-page application handles all UI state — screens, conversation history, evaluation results. All Claude API calls are proxied through Netlify serverless functions, which keeps the API key secure server-side.

Prompt design

The invisible scaffolding principle

The hardest part wasn't the React code — it was writing a system prompt that produces genuinely Socratic behaviour. The coach had to feel warm but never sycophantic, rigorous but never lecturing, and personal in the Personal Decisions mode without drifting into therapy.

The key design decision was making the 5 CT components invisible scaffolding. The coach doesn't label what it's doing ("Now let's evaluate your evidence…") — it just asks the questions that naturally surface each component. The evaluation is the only moment those components become visible, and only in retrospect.

I also built two separate system prompts — one for scenario mode, one for personal decisions — because the stakes and tone required meaningfully different behaviour. In Personal mode, the coach can name avoidance patterns and push on rationalisation more directly. In scenario mode, it stays more analytically neutral.

Reflection
What this taught me

Building this clarified something important about how I want to use AI in my work: the most useful AI tools are often the ones that make you do the thinking, not the ones that do it for you. That's a different design brief than most AI products operate from.

Technically, the most interesting challenge was the serverless function setup — specifically getting the dependency bundling right for Netlify's build pipeline, and understanding the difference between esbuild's module resolution and Node's native module resolution at runtime.

The prompt engineering surprised me. Getting consistent Socratic behaviour across wildly different scenarios and difficulty levels required more iteration than the code did. A system prompt is a kind of design artefact — it has to handle edge cases, establish a tone, and constrain behaviour without making the output feel constrained.
